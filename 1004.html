Errata for Internationalization and Localization Using Microsoft .NET

 

CorVars.bat

This book was written using Beta 2 and the first release version of the Framework.  This section covers corrections using Microsoft.Net 2003.  This is the latest version of .NET.

 

Throughout the book I use command line programs to create resource files and DLL’s.  In each of these cases I tell the user to run a batch file called CORVARS.BAT.  Microsoft supplied this batch file up until the latest version of .NET.   This batch file does not exist for .NET 2003. 

 

To get the correct DOS environment setup for command line utilities, you will need to run the “Visual Studio .NET 2003 Command Line Prompt” from the Windows Start menu.  This provides a command prompt with the correct environment variables.

 

 

Chapter 10 pages 308-313

Page 312:

The “al” command that is used to compile the ch10snvb.resources file into a DLL with delayed signing is incomplete.  It is missing the letters “snk” at the end of the command.  The command should be: al /out:CH10SNVB.Resources.Dll /v:1.0.0.0 /c:en-US /embed:CH10SNVB.en-US.resources,CH10SNVB.en-US.resources /delay+ /keyfile:pubkey.snk

 

Page 312:

The text says to use the sn –Vr ch10snvb.resources.dll command followed by the gacutil –i ch10snvb.resources.dll command.

 

The “sn” command disables version checking of the resource DLL so the “gacutil” command can place the resource file in the global assembly cache.

 

After the “gacutil” command you will need to type in another command: sn –Vr *

 

This last “sn” command registers the resource file you put in the GAC to require no verification when your program tries to use it.  This “sn” command is vital to the getting this example to work.  Without it you will get an error stating that the resource file could not be verified.  It is this command that is missing from the book.

 

Page 313:

I tell you to include two assembly directives that allow you to get access to the resource file you placed in the GAC.  The reason you need to do this is to synchronize the program strong name with the resource file strong name.  They must both be signed with the same public key.
